## 1. Source Code Submission Guide

All teams participating in the final round **must**, **when submitting algorithms from October 14**, compress all source code and readme.txt file together according to the guidelines below and submit (no need to create separate subdirectories).

- **If there are library files that are pre-compiled and used, such as Shared Object files**, you **must submit source code before compilation** and **create a separate plain text readme.txt file describing how to compile**. However, library files provided in the OGC2024 environment cannot have source code uploaded, so those source codes may be excluded.
- **If there are no pre-compiled library files such as Shared Object files but external packages not provided in the OGC2024 environment are used, you must submit source code and describe the packages in the readme.txt file**. **If it is a publicly accessible external package but source code submission is impossible, you must describe the package and access method**.
- **If all files submitted to the system are source code and there are no problems running them in the OGC2024 environment as-is**, you can **briefly write that fact in the readme.txt file and submit**.
- **Warning**: After the final round ends on the 16th, the organizing committee will review the last submitted source code and check if reproduction works through compilation, etc., and please note that disqualification may occur if problems arise. As announced during competition participation, source code will be publicly released to everyone.

## 3. Final Presentation Material Preparation

Presentation time per team is limited to within 10 minutes. Prepare presentation material file format in PowerPoint or PDF, freely including the following items:

- **Algorithm Logic**: Explain overall algorithm structure and details
- **Algorithm Implementation**: Explain algorithm implementation language, packages used, implementation techniques, etc.
- **Algorithm Strengths**: Explain what the team wants to emphasize about the algorithm (logic + implementation) (e.g., creativity, applicability in various situations, scalability according to problem size, etc.)
- **Algorithm Development Direction**: Introduce algorithm future improvement directions, methods not used due to technical or competition period constraints
- **Competition Participation Review**: Impressions, suggestions, competition future development directions, etc.

> ðŸ’¡ **Presentation time per team is limited to within 10 minutes**, so **include the above items (logic, implementation, strengths, development direction, participation review)** while considering the time limit when preparing presentation materials

## 4. Final Presentation Evaluation Criteria and Final Score Calculation

Final presentation evaluation is conducted by a judging panel of 2 industry members and 2 academia members evaluating the following items:

- **Algorithm Appropriateness**: Is the algorithm logic appropriate and effective for solving the problem? Is it applicable to various situations such as problem data changes? Evaluated
- **Algorithm Creativity**: Algorithm originality, novelty of approach method, etc. evaluated
- **Algorithm Efficiency**: Use of techniques or implementation technologies to increase algorithm execution efficiency, scalability according to problem size, etc. evaluated
- **Presentation**: Presentation material volume, expression, content delivery, Q&A response, etc. evaluated

The judging panel calculates team presentation evaluation scores on a 25-point scale. Team final scores are calculated by combining leaderboard scores (100 points maximum) and presentation evaluation scores (25 points maximum) (80% : 20%). According to final score ranking, 1 grand prize team, 1 best excellence prize team, 2 excellence prize teams, 3 special prize teams, and 3 encouragement prize teams are determined in order, and winners will be announced at the awards ceremony on October 25 (Friday).
