# 결선 관련 참고 사항

## 1. 연습용 추가 문제

결선을 위한 추가 연습문제는 없습니다. 본선에서 사용한 연습문제들과 결선에서 사용될 평가문제들은 특성이 비슷합니다. 참고로 결선 평가문제들은 노드 개수 기준으로 최대 400개가 넘지 않을 예정입니다.

## 2. 알고리즘 시간제한

결선의 알고리즘 실행시간 제한은 문제마다 다르게 주어집니다. **각 평가 문제마다 제한시간이 주어지고 (**`algorithm()` 함수의 `timelimit` 파라메터**) 제한시간의 범위는 최소 0.5분(30초, `timelimit=30`)에서 최대 10분(600초, `timelimit=600`)까지 입니다.** 알고리즘이 실행시간 제한을 넘기면 문제풀이를 강제로 종료하고 그 문제에 대해서는 최하 점수(-1점)을 받습니다. 각 팀별 득점 산출 방식은 예선과 동일합니다.

> 💡 알고리즘의 내부에서 지속적으로 실행시간을 체크하여 제한시간을 넘지 않아야 함. 예를 들어 알고리즘 내부에서 사용하는 특정 함수의 시간이 오래걸릴 수 있는 경우 그 함수 내부에서 제한시간을 체크하지 않는다면 알고리즘의 실행 시간이 제한시간을 넘길 수 있음. 참가팀은 이런 부분에 유의하여 알고리즘이 제한 시간을 넘지 않도록 할고리즘의 실행단계 마다 충분한 여유시간이 있는지 확인해야 함


> 💡 참가팀이 알고리즘을 개발하는 환경과 평가 서버의 환경 차이로 발생할 수 있는 실행 시간차이도 주의필요. 특히 같은 알고리즘을 같은 서버 환경에서 실행하더라도 알고리즘 실행시간에는 약간의 차이가 발생할 수 있는데, 이점을 고려해야 함


## 3. 결선 진행 방법

결선은 추가적인 알고리즘 제출 없이 **본선에서 제출한 마지막 알고리즘으로 진행** 됩니다. 하루에 두 문제씩 7일 동안 총 14문제를 풀게 됩니다. 문제를 푼 결과는 매일 12시, 24시에 한 문제씩 추가하여 리더보드로 공개합니다. 순위는 리더보드 공개 시점까지 푼 문제들을 누적해서 부여한 점수로 정해집니다. 결선까지 온 팀들은 그동안 고생 많이 하셨으니 느긋하게 매일 어떻게 순위가 바뀌는지 지켜보세요~


## 4. 소스코드 제출 안내

결선에 참여한 모든 팀은 **아래의 요령에 따라 모든 소스 코드와 readme.md 파일** 을 같이 압축하여 제출하기 바랍니다.

- **제출한 소스 코드는 본선에서 마지막에 제출한 코드와 기본적으로 동일해야 합니다.** “기본적으로 동일함”의 의미는 알고리즘을 실행했을 때 같은 결과를 얻는것을 말합니다. 버그 수정이나 코드 개선은 절대 불가합니다. 본선 마지막 제출(결선에서 사용되는 알고리즘)을 그대로 제출하되 소스에 대한 설명 파일(readme.md)과 라이브러리에 대한 소스 파일만을 추가합니다.
- **Shared Object 파일 등 미리 컴파일 되어 사용되는 라이브러리 파일이 있을 경우** 에는, **반드시 컴파일 되기 전의 소스 코드를 제출** 해야 하고 별도의 plain text 파일인 **readme.md 파일을 만들고 컴파일 하는 방법을 기재** 하기 바랍니다.
- **Shared Object 파일 등 미리 컴파일 되어 사용되는 라이브러리 파일은 없으나 OGC2025 환경에서 제공되지 않는 외부 패키지를 사용하는 경우,  소스 코드를 제출하고 readme.md 파일에 패키지 설명을 기재** 해야 합니다. **만일 누구가 엑세스할 수 있는 공개된 외부 패키지인데 소스코드 제출이 불가하다면, 패키지 설명과 엑세스 방법을 기재** 해야 합니다.
- **시스템에 제출하는 파일들이 모두 소스코드이고 그 자체로 OGC2025 환경에서 구동되는 데 문제가 없다면**, **`readme.md` 파일에 간략이 그런 사실을 쓰셔서 제출** 하시면 됩니다.
- `readme.md` 파일에는 그 밖에 참가팀이 원하는 추가 설명이 포함될 수 있습니다.
- **주의** : 결선이 진행되는 동안 운영위원회에서 제출된 소스 코드를 검토하고, 컴파일 등을 통해서 재현이 그대로 되는지 체크할 예정이며, 문제가 발생할 시에 실격 처리할 수 있음을 유념해 주십시오. **대회 참가시에 고지된 바와 같이 소스코드는 모두에게 공개될 예정입니다.**
- 소스코드와 모든 설명을 정해진 파일이름(**팀명_소스.zip)**으로 압축해서 아래 링크로 업로드(**9월 5일까지!**)

## 5. 결선 발표 준비

결선 참가팀은 모두 9월 19일 발표 평가를 준비해야 합니다.

발표 평가는 팀 별로 **5분 발표, 5분 질의 응답**으로 진행됩니다. 

5분 발표를 위한 동영상을 미리 제작해야 합니다. (**현장에서는 제출한 동영상을 재생하고 질의 응답을 진행합니다.**)

5분 발표 동영상은 다음의 내용을 포함하여 자유롭게 작성하세요.

- **알고리즘 로직** : 알고리즘 전체적인 구조와 세부사항을 설명
- **알고리즘 구현** : 알고리즘 구현 언어, 사용 패키지, 구현 기술 등을 설명
- **알고리즘 특장점** : 팀별로 알고리즘(로직 + 구현)에 대해 강조하고 싶은 내용을 설명 (예. 창의성, 다양한 상황에서의 적용성, 문제 크기에 따른 Scalability 등)
- **알고리즘 발전 방향** : 알고리즘의 추후 개선 방향, 기술적인 혹은 경진대회 기간의 제약 등으로 사용하지 못한 방법이 있다면 소개
- **경진대회 참여 후기** : 소회, 건의 사항, 경진대회의 추후 발전 방향 등
- **동영상 해상도(HD 이상) 및 음성 및 소리가 명확하게 재생되는지 꼭 확인!**
